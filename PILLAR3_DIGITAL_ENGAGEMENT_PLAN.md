# SBI Pulse 🫀 — Agentic Proactive Engagement Brain for YONO
### Problem Statement · Pillar 03 · Digital Engagement (only)

> **Pillar 03:** *"Create AI-driven engagement models that proactively interact with
> customers based on **behaviours, financial patterns, and life events**."*
> Sub-goals it names: **Personalised customer engagement · Intelligent financial
> interactions · AI-led banking journeys.**

**SBI Pulse** turns SBI's ~100M passive account-holders into an actively engaged,
financially-healthier base — by continuously sensing each customer's behaviour, financial
patterns, and life events, then **autonomously orchestrating personalised, multi-step
engagement *journeys*** across channels, with **financial wellness as the north star** and
**consent + explainability + restraint** built in.

**Theme:** Agentic AI & Emerging Tech. **Scope:** Digital Engagement only.

---

## 0. The core bet (read first)

Most teams will answer "engagement" with a **chatbot** or a **single product-recommendation
nudge**. That is the 2020 answer. The 2026 answer — and the one that wins — is:

> **Stop sending nudges. Run journeys.**

An agent that operates at the **journey level** — anticipating needs before they're
expressed, initiating a multi-step conversation, adapting to how the customer responds,
choosing the right channel and moment, and *knowing when to stay silent* — is the
differentiated build. Gartner ties exactly this (agentic journey orchestration) to a 35%
performance edge by 2026. Everything below is engineered around that bet.

## 2. The product — three engines, one brain

**SBI Pulse** is composed of three engagement engines that map 1:1 to the pillar's own goals:

```
                         ┌──────────────  SBI PULSE  ──────────────┐
                         │        the proactive engagement brain    │
   behaviours ──▶        │                                          │
   financial   ──▶       │  ① ANTICIPATORY ENGINE                   │
   patterns             │     life events + pattern shifts →        │  ← Pillar: "behaviours,
   life events ──▶       │     "what's happening in this life?"     │     financial patterns,
                         │                                          │     life events"
                         │  ② FINANCIAL-WELLNESS ENGINE             │  ← Pillar: "intelligent
                         │     spend/save/goal coaching, distress    │     financial interactions"
                         │     detection, gamified habit-building    │
                         │                                          │
                         │  ③ JOURNEY ORCHESTRATOR                  │  ← Pillar: "AI-led banking
                         │     multi-step, adaptive, cross-channel,  │     journeys" + "personalised
                         │     right-time, frequency-capped          │     customer engagement"
                         └──────────────────────────────────────────┘
```

- **① Anticipatory** — infers life events (new job, relocation, new baby, retirement,
  **financial stress**) *and* behavioural-pattern shifts (spend spikes, income dips, dormancy,
  goal drift). Answers: *what changed in this customer's life?*
- **② Financial-Wellness** — turns signals into genuinely helpful interactions: spending
  insights, savings/goal coaching, overspend rescue, distress support, and **gamified
  habit-building** (streaks, goals, milestones — aligned to YONO 2.0). Answers: *how do we
  help, not just sell?*
- **③ Journey Orchestrator** — the agentic core. Instead of firing one nudge, it launches a
  **multi-step journey**, picks the channel + moment, adapts to the customer's response, and
  respects consent/frequency. Answers: *what's the next best engagement, right now, for this
  person?*

## 3. "Journeys, not nudges" — the concept made concrete

A **nudge** = one message. A **journey** = an adaptive, multi-step, goal-driven engagement the
agent runs over days/weeks, branching on behaviour. Examples (each spans behaviours /
patterns / life events):

| Journey | Trigger (signal) | Steps (agent adapts) | Wellness goal |
|---|---|---|---|
| **First SIP** | Salary hike / stable surplus | Insight → tiny-start offer → reminder → celebrate first SIP → step-up | Build investing habit |
| **Overspend Rescue** | Spend velocity ↑, balance ↓ | Gentle heads-up → categorised insight → set a soft cap → weekly check-in | Regain control |
| **Dormant Revival** | No activity 60–90d | Re-intro → one-tap useful action (bill/UPI) → streak start → reward | Reactivate the 34% |
| **New-Baby Nest** | Hospital→baby-care pattern | Congrats → child-goal RD → protection review (→ human RM) | Plan for the child |
| **Stress Shield** | Income dip + min-payment + EMI | **Suppress cross-sell** → restructuring help → budget coach → follow-up | Support, not sell |
| **Retirement Runway** | Age + pension inflow | Safety-first insight → SCSS/annuity (→ RM) → income plan | Secure income |

The agent decides **which journey, which step, which channel, what time, and whether to speak
at all** — that is the "AI-led banking journey" the pillar asks for.

```
 SIGNALS      behaviours · transactions · financial patterns · life events · app usage · dormancy
    │

 ┌─────────────────────────  SIGNAL & FEATURE LAYER  ─────────────────────────┐
 │  event inference · pattern-shift detection · engagement + wellness features │
 └───────────────────────────────────────┬────────────────────────────────────┘
                                          ▼
 ┌────────────────────────────  ENGAGEMENT BRAIN  ────────────────────────────┐
 │  customer engagement STATE model · churn-risk · propensity · financial-     │
 │  health score  →  NEXT-BEST-ENGAGEMENT decision (which journey / step)      │
 └───────────────────────────────────────┬────────────────────────────────────┘
                                          ▼
 ┌───────────────────────────  JOURNEY ORCHESTRATOR  ─────────────────────────┐
 │  journey state machines · step selection · channel + timing + frequency ·   │
 │  adapts to customer responses · A/B + uplift experimentation                │
 └───────────────────────────────────────┬────────────────────────────────────┘
                                          ▼   every send passes through ↓
 ┌────────────────────────  GUARDRAIL ENGINE (deterministic)  ─────────────────┐
 │  DPDP consent · TRAI 9–9 window · frequency cap · distress-suppression ·     │
 │  human-in-loop for high-value · fatigue control                             │
 └───────────────────────────────────────┬────────────────────────────────────┘
                                          ▼
 ┌──────────────────  DELIVERY  ──────────────────┐   ┌──────  MEASUREMENT + LEARNING  ──────┐
 │  YONO in-app cards · WhatsApp · push · email    │   │  engagement lift · journey funnels ·  │
 │  · gamified widgets (streaks/goals)             │   │  churn↓ · uplift · feedback→thresholds│
 └─────────────────────────────────────────────────┘   └───────────────────────────────────────┘
                                          │
              EXPLAINABILITY LEDGER (append-only): who · what journey/step · why · consent basis
```

**Design rules (credibility):** the guardrail engine is **deterministic code, not the LLM**;
every engagement is **logged with its "why" and consent basis**; the agent **optimises a
wellness-aware objective**, not raw conversions.

## 6. Engagement intelligence (the metrics the agent optimises)

- **Engagement Score** (0–100) per customer — recency/frequency/breadth of meaningful actions.
- **Financial-Health Score** — balance trend, savings rate, credit stress, product depth.
- **Churn-risk** and **dormancy-risk** — trigger revival journeys early.
- **Propensity / next-best-engagement** — which journey has the highest *helpful* impact.
- **Uplift, not clicks** — measure incremental behaviour change vs. a holdout, so we reward
  genuine engagement, not spam. (This is the anti-dark-pattern guarantee, quantified.)

---

## 7. Responsible engagement (the moat)

- **Consent-native (DPDP 2023 / Rules 2025)** — purpose-scoped, revocable; AA-only external data.
- **Restraint by design** — distress detection **suppresses cross-sell**; wellness-first objective.
- **Frequency + fatigue caps**, **TRAI 9am–9pm** window, one-tap opt-out.
- **Explainable** — "Why am I seeing this?" on every touch + full audit ledger.
- **Human-in-the-loop** for high-value actions (loans/insurance) — agent briefs, human decides.

## 8. PHASED PLAN (Pillar 3 only)

### Phase 0 — Hackathon MVP (48–72h) · *win the room*
**Goal:** a working engagement brain that runs **live, adaptive journeys** on synthetic
personas, with the Ops dashboard proving impact — offline-safe, no API key required.

**Build:**
- **Signal + Anticipatory + Wellness engines** (reuse & extend the existing engine): life
  events **+** pattern shifts + engagement/health scores.
- **Journey Orchestrator** with **5 journeys** (First SIP, Overspend Rescue, Dormant Revival,
  Stress Shield, New-Baby Nest) as adaptive state machines that branch on customer response.
- **Deterministic guardrails + explainability ledger** gating every step (reuse existing).
- **Customer view:** YONO-style feed where a journey unfolds step-by-step; the customer can
  reply and advance/branch it; gamified streak/goal widget.
- **Ops "Engagement Cockpit":** engagement-score distribution, active journeys + funnels,
  churn-risk board, suppression/compliance counter, **projected retention uplift**.

**Tech:** existing TS/Node engine · Claude (optional) for message tone + free-form replies,
with template fallback · zero-dep web server + hand-built UI (already have the spine).

**Success = judges watch a real multi-step journey adapt to a customer, see the agent stay
silent on a stressed customer, and see the dashboard quantify retention uplift.**

---

### Phase 1 — Pilot (Month 1–3) · *prove engagement lift*
- Integrate a **real event/stream** (YONO events + **AA** consented data) behind flags.
- Ship **3 journeys** to a consented cohort (staff + volunteers), **holdout group** for uplift.
- Stand up **measurement infra first** (engagement score, funnels, uplift) before scaling.
- Channels: YONO in-app + WhatsApp. Human-in-loop on all high-value steps.
- **KPIs:** engagement-score lift vs holdout, dormant-revival rate, opt-out rate (low = healthy),
  journey completion, products/customer movement.

---

### Phase 2 — Rollout (Month 3–9) · *scale journeys*
- **10–15 journeys**; add push, email, and **gamification layer** (streaks, goals, rewards —
  wired to YONO 2.0's gamification direction).
- **Vernacular** engagement (Bhashini) for inclusion; right-time send-time optimisation.
- Full **DPDP consent-manager** + **TRAI DLT** compliance; fatigue/frequency governance at scale.
- Uplift-based journey optimisation (retire journeys that don't help).
- **KPIs vs SBI's own numbers:** churn↓ (target 15–25%), products/customer↑ (from 2.3),
  dormancy↓ (from 34%), MAU/engagement↑.

---

### Phase 3 — Platform (Month 9–24) · *own engagement*
- **Journey Studio** — business users compose/govern journeys without code.
- **RM cockpit** — the engagement brain as a next-best-engagement copilot for relationship managers.
- Open **skills** (govt schemes, farmer/MSME engagement), edge/offline for low-connectivity.
- Continuous-learning loop with model governance, drift monitoring, fairness audits.

## 10. Demo script (3 minutes)
1. **A journey adapts (the hook):** persona "Neha" gets a *First SIP* journey — insight →
   tiny-start → she replies "not now" → the agent **branches** to a smaller goal, not a nag.
2. **Anticipation:** "Ravi" relocates + gets a raise → the agent opens a relevant journey at
   the right moment, in his language.
3. **Restraint (the memorable beat):** "Meena" is in financial stress → the agent
   **suppresses the cross-sell** and runs the *Stress Shield* care journey instead.
4. **Engagement Cockpit:** flip to the dashboard — engagement scores, live journey funnels,
   churn-risk, compliance suppressions, and **projected retention uplift**.
5. Close on the four differentiators: **journeys not nudges · anticipatory · wellness-first &
   knows when not to sell · consent + explainable by design.**

## 11. Risks & mitigations

| Risk | Mitigation |
|---|---|
| "It's just nudges" | Show a journey *branch* live on a customer reply — that's the agentic differentiator |
| Creepy / dark-pattern feel | Wellness-first objective, uplift (not clicks) as the metric, restraint on distress, one-tap opt-out |
| Engagement fatigue / spam | Deterministic frequency + fatigue caps, TRAI window, holdout-measured uplift |
| Regulatory (DPDP/TRAI/RBI) | Consent-native, AA-only external data, DLT-registered outbound, full audit from day one |
| False-positive signals | Confidence thresholds, human-in-loop for high value, "was this useful?" feedback loop |
