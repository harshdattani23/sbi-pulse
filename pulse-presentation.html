The solution — how we solve it

Stop sending nudges. Run journeys.

📡
Sense. Per-customer signals from the transaction stream — life events (new job, relocation, baby, retirement), pattern shifts (overspend, income dip), dormancy.
✨
Reason (real AI). A live AI agent reads the actual data, infers the situation, and decides the best engagement — or that silence is the right move.
💬
Engage. A multi-step conversation in the customer's language that adapts to every reply — “not now” gets a gentler path, never a nag.
🛡️
Restrain. On detected financial stress the agent suppresses all cross-sell and leads with help. Trust is the product.
📈
Learn. Every outcome trains a bandit policy — the system gets measurably better with use.

🔍 Explainability audit trail
Every decision logs observe → infer → decide → gate → deliver → reply → learn with the reason and consent basis. An auditor can replay any engagement.
⚖️ Why this split matters
The model can never bypass consent, contact windows, or caps — those live in plain, reviewable code. This is the 2026 best practice for agentic banking (“architecture before agent”).
The AI core — genuinely agentic

The AI reads the actual transactions and decides

🧠 Situation inference
“Meena is managing high fixed expenses like rent and EMIs, with low savings and indicators of financial stress.” — the model's own read of her real statement, not a rule's output.
🎯 Autonomous decision
Chooses the journey with confidence + written rationale — including “none — stay silent” when engagement wouldn't help. Restraint is a decision, not an accident.
🗣️ Generated vernacular
Messages written fresh in Hindi, Odia, or English per the customer — type an English reply into a Hindi conversation and it answers in Hindi.
💬 Free-text conversation
A real chat, not buttons: the agent responds in-character, decides when the journey completes (positive / declined / human hand-off), and hands high-value products to an RM.
Lightweight production LLM · structured JSON reasoning · ~1.5s per call · model-agnostic (swappable to an India-hosted / Banking-BHASHINI model) · automatic rule-engine fallback so the demo never breaks.

The moat

The demo is reproducible.
The flywheel isn't.

🔁
Outcome feedback flywheel. A Thompson-sampling policy learns which tone works per segment from real outcomes — live journeys apply the learned tone and feed results back. Using the product trains it.
📊
Verified learning. From ≈random start to +12.9pp over random, 91% of the theoretical ceiling — and it correctly discovers the hidden best tone for every segment.
🏦
Only compounds at SBI's scale. Converging the policy needs a stream of consented outcomes only a ~100M-customer base produces. A competitor can clone the UI; they can't clone the data flywheel.
⚖️
Compliance lock-in. Consent, TRAI, audit and human-in-loop wired in from day one — the months-long certification work that blocks every would-be replacement.

Roadmap — demo to platform

Phased, measurable, de-risked

Phase 0 · now
Working MVP. Live AI journeys, 5 surfaces, guardrails, vernacular, audit trail, uplift measurement, learning flywheel.
Phase 1 · 1–3 mo
Pilot. Consented Account Aggregator + YONO event streams; real holdout uplift on a volunteer cohort; human-in-loop on all high-value.
Phase 2 · 3–9 mo
Rollout. Push/IVR channels, 10–15 journeys, gamification tie-in with YONO 2.0, Bhashini vernacular, DLT registration.
Phase 3 · 9–24 mo
Platform. Journey Studio for business users, RM copilot, 22 languages, edge/offline, model governance & drift monitoring.
Each phase gates on its own KPIs — engagement lift vs holdout, dormant-revival rate, opt-out rate (low = healthy), products per customer.
