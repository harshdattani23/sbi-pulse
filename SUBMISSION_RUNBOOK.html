1

Prerequisites

One tool. No package install — the project is zero-dependency and runs TypeScript natively.
  1. Install Node 22 or newer.
    Check what you have; if it's missing or old, install it.
    node --version            # need v22+ (v25 is fine)
# if missing, on macOS:
brew install node
  2. Get the project folder.
    Everything lives in the sbi directory. Open a terminal there.
    cd path/to/sbi
ℹ️
No npm install is needed — there are no dependencies. Node executes the TypeScript directly via --experimental-strip-types, already wired into the npm scripts.
2

Add your Gemini key

The reasoning + conversation run on Google Gemini. The key is read only from a git-ignored .env file.
  1. Create a fresh API key.
    Go to aistudio.google.com/apikey, sign in, and create a key (free tier is plenty). Model used: gemini-3.1-flash-lite-preview
  2. Put it in .env. 
    echo "GEMINI_API_KEY=your-fresh-key-here" > .env
🔑
Rotate the old key. The key used during development was shared in chat — treat it as compromised. Delete it in AI Studio and use a new one for the recording.
✅
No key? The app still runs — it falls back to the deterministic rule engine (the AI badge just won't show). For the video, make sure the key works so the real Gemini reasoning is on.

The demo — what to show & say

≈3 minutes. This is the spine of the video. Narrate each beat in your own words.
  1. Frame it (15s).
    Open the deck at /deck.html (F fullscreen, → to advance). Slides 1–3: the problem (34% dormant, 2.3 products/customer) and the thesis — “journeys, not nudges.”
  2. Real AI + restraint (45s).
    Switch to the app → Journey tab → click Meena. Say: “Gemini read her actual transactions.” Point to the inferred situation and the audit trail (observe → infer → decide → gate → deliver). It detected stress, chose care over a sale, and wrote the message in Hindi.
  3. It's a real conversation (25s).
    Type a reply into the phone (e.g. “yes please help me”). It responds live, in her language — not fixed buttons.
  4. It adapts (25s).
    Click Neha (a raise → First SIP). Reply “not now.” Show it doesn't nag — it eases off. Point to the scores panel on the right.
  5. Proof, not vibes (25s).
    Impact tab: “+20pp uplift versus a randomised holdout — measured on outcomes, not clicks.”
  6. The moat (30s).
    Flywheel tab: “A learning policy that improves from every outcome — and the live journeys feed it.” Click Run 2,000 more twice; watch the blue line chase the ceiling.
  7. Platform + close (15s).
    Journey Studio tab: “every journey is a governed, no-code state machine.” Close on the four differentiators (below).
▶
Hands-free option: the top-bar Guided demo button auto-walks the hero beats with captions — great as a backup take or if you'd rather not click live.
🎯
The four lines to land: journeys not nudges · knows when not to sell · consent + explainable by design · learns from every outcome (the moat).
5

Record the video (macOS)

Clean screen, narrated, under the time limit. Two takes.
  1. Prep the stage.
    Turn on Do Not Disturb, hide the browser bookmark bar, close other tabs, tidy the desktop, browser to fullscreen (⌃⌘F).
  2. Start recording.
    Press ⇧⌘5 → choose Record Entire Screen (or a window) → enable your microphone for narration → Record. (QuickTime → New Screen Recording works too.)
  3. Run the Phase 4 script.
    Speak steadily; don't rush the Meena/restraint beat — it's the memorable one.
  4. Do a second take and also capture one Guided demo run as an automated fallback.
  • Trim & export.
    Trim in QuickTime (⌘T) or iMovie; export at 1080p. Aim for the hackathon's limit (typically 3–5 min).
  • Upload & get a link.
    YouTube (Unlisted) or Google Drive (link-sharing on). Test the link in a private window.

    Pillar03 — Digital Engagement (Agentic AI & Emerging Tech).
    Copy-paste solution summary

    SBI Pulse is an agentic AI engagement platform for Pillar 3. It senses each customer's behaviours, financial patterns and life events, and runs proactive, adaptive engagement journeys — not one-off nudges.

    A live Gemini agent reads the actual transaction stream, infers the situation, decides the best journey (or to stay silent), and holds a free-text conversation in the customer's language (Hindi / Odia / English). A deterministic guardrail layer enforces DPDP consent, the TRAI contact window and frequency caps — the LLM proposes, policy disposes — and every decision is logged in an explainability audit trail. On detected financial stress it suppresses cross-sell and offers care.

    A Thompson-sampling learning policy improves message-tone selection from real outcomes, and the live journeys feed it — so the product gets better with use. Impact is measured as uplift against a randomised holdout (+20pp), not clicks.
